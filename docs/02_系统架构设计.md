# Agent Workbench 系统架构设计

## 1. 架构目标

系统架构要服务三个核心要求：

```text
1. 多智能体任务可并行执行
2. 每个执行过程可实时可视化
3. 所有过程可持久化、可回放、可调试
```

因此系统必须以事件流为中心，而不是以最终回答为中心。

## 3. 技术选型

### 3.1 桌面端

推荐：Tauri 2 + React + TypeScript。

原因：

```text
体积小
性能好
适合本地应用
可调用系统能力
可用 Rust/Tauri 管理安全边界
```

如果开发团队更熟悉 Node/Electron，也可以用 Electron，但本方案默认 Tauri。

---

### 3.2 前端

```text
React
TypeScript
Vite
Zustand
React Flow，P1
CSS Animation / Framer Motion
```

前端职责：

```text
订阅事件流
用 reducer 推导 UI 状态
展示 Agent 卡片
展示并行泳道
展示 Trace Console
展示 Inspector
发送用户控制命令
```

前端不负责：

```text
任务调度
权限判断
API Key 解密
直接调用 LLM
```

---

### 3.3 后端

推荐 Node.js TypeScript sidecar。

原因：

```text
LLM SDK 支持成熟
流式处理方便
工具系统开发快
与前端类型可共享
```

后端职责：

```text
任务创建
总智能体规划
DAG 调度
Agent 执行
工具调用
事件生成
事件持久化
Provider 适配
权限控制
```

---

### 3.4 数据存储

```text
SQLite：存 session、tasks、events、agents、settings
OS Keychain：存 API Key 明文
SQLite 只存 api_key_ref
```

## 4. 运行时模块

### 4.1 Orchestrator

职责：

```text
接收用户任务
调用总智能体生成任务计划
校验任务计划结构
创建 task graph
在任务完成后调用总智能体汇总最终结果
```

不负责：

```text
逐个管理 Agent 运行细节
直接调用工具
直接操作 UI
```

---

### 4.2 Scheduler

职责：

```text
维护任务 DAG
判断任务依赖
找出可运行任务
并行启动 AgentRuntime
处理任务完成/失败
处理取消/暂停
控制最大并发数
```

MVP 并发建议：

```text
默认最大并发 3
可配置最大并发 1-5
```

---

### 4.3 AgentRuntime

职责：

```text
执行一个 AgentTask
读取 Agent 配置
读取依赖任务结果
构造模型请求
处理模型流式输出
处理工具调用
持续 emit 事件
返回结构化任务结果
```

每个正在执行的任务对应一个 AgentRuntime 实例。

---

### 4.4 ToolRuntime

职责：

```text
注册工具
校验工具输入
执行工具
记录工具输出
处理工具错误
发出 tool_call_started / tool_call_finished / tool_call_failed 事件
```

MVP 工具：

```text
web_search，P0 为 mock 实现，P1 切换真实 provider
text_summarize，内部 LLM 可选
file_read，P1
file_write，P2
terminal，P2
browser，P2
```

---

### 4.5 EventBus

职责：

```text
接收所有内部事件
写入 TraceStore
推送给前端
支持 session 级订阅
支持历史事件重放
```

EventBus 是系统核心。

事件流必须满足：

```text
有序
可持久化
可重放
可用于恢复 UI 状态
```

---

### 4.6 TraceStore

职责：

```text
将事件写入 SQLite
events 表追加写
按 session 查询事件
按 task 查询事件
按 agent 查询事件
```

MVP 中 events 表是最重要的数据表。

---

### 4.7 ProviderAdapter

职责：

```text
屏蔽不同模型供应商差异
统一流式输出事件
统一工具调用格式
统一 token/cost 统计
统一错误格式
```

接口示意：

```ts
interface ProviderAdapter {
  streamMessage(input: ProviderRequest): AsyncIterable<ProviderStreamEvent>;
  createMessage(input: ProviderRequest): Promise<ProviderResponse>;
}
```

## 5. 数据流

### 5.1 创建任务数据流

```text
用户输入任务
↓
Frontend 调用 POST /sessions
↓
Backend 创建 session
↓
EventBus emit session_started
↓
Orchestrator 调用总智能体规划
↓
EventBus emit plan_created
↓
Scheduler 创建任务 DAG
↓
Frontend 根据事件显示任务和 Agent 状态
```

---

### 5.2 Agent 执行数据流

```text
Scheduler 找到可执行 task
↓
启动 AgentRuntime
↓
AgentRuntime emit task_started
↓
AgentRuntime 调用 ProviderAdapter
↓
模型 text delta -> emit agent_message_delta
↓
模型请求工具 -> ToolRuntime 执行
↓
emit tool_call_started / finished
↓
AgentRuntime 得到最终结果
↓
emit task_finished
↓
Scheduler 检查下游任务
```

---

### 5.3 最终汇总数据流

```text
所有任务完成
↓
Scheduler 通知 Orchestrator
↓
Orchestrator 收集所有 task result
↓
调用总智能体汇总
↓
emit final_summary_started
↓
流式输出 final_answer_delta，P1
↓
emit session_finished
↓
Frontend 展示最终结果
```

## 6. 状态分层

### 6.1 持久状态

存储在 SQLite：

```text
agents
sessions
tasks
events
settings
```

### 6.2 运行时状态

保存在后端内存：

```text
active sessions
running tasks
cancellation tokens
provider streams
tool executions
```

### 6.3 UI 状态

保存在前端 Zustand：

```text
currentSession
agents
tasks
events

selectedTaskId
selectedEventId
activeWorkbenchView
```

UI 状态应能从 events 重建。

## 7. 模块依赖关系

```text
Frontend
  depends on API + Event Stream

Orchestrator
  depends on ProviderAdapter + EventBus + TaskStore

Scheduler
  depends on TaskStore + AgentRuntime + EventBus

AgentRuntime
  depends on ProviderAdapter + ToolRuntime + EventBus + TaskStore

ToolRuntime
  depends on ToolRegistry + PermissionManager + EventBus

EventBus
  depends on TraceStore + RealtimeGateway

TraceStore
  depends on SQLite
```

## 9. 关键架构约束

### 9.1 前端不直接调用 LLM

所有 LLM 请求必须经过后端 ProviderAdapter。

原因：

```text
保护 API Key
统一事件记录
统一错误处理
统一 token/cost 统计
统一工具权限控制
```

---

### 9.2 工具调用必须走 ToolRuntime

Agent 不能绕过 ToolRuntime 直接执行本地能力。

原因：

```text
权限控制
日志记录
用户确认
失败恢复
安全边界
```

---

### 9.3 所有状态变化必须 emit 事件

如果某个状态变化没有事件，前端就无法可靠显示，也无法回放。

```text

GET /sessions/:id/events
WebSocket/SSE 事件推送
Orchestrator 规划
Scheduler 基础 DAG

ClaudeProvider 基础流式调用
```

```text

真实终端控制
复杂权限 UI

云同步
```
