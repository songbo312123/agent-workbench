# Agent Workbench 数据模型与数据库设计

## 1. 数据设计目标

MVP 使用 SQLite，目标是：

```text
1. 保存 Agent 配置
2. 保存任务会话
3. 保存任务 DAG
4. 保存完整事件流
5. 支持历史查看和回放
6. 支持从事件恢复 UI 状态
7. 避免明文保存 API Key
```

---

## 2. 核心实体

```text
Agent：智能体配置
ApiKeyRef：API Key 引用
Session：用户的一次任务会话
Task：会话中的一个子任务
Event：所有执行轨迹事件
ToolCall：工具调用记录，可从 events 派生，也可独立存储，P1
Setting：用户设置
```

实体关系：

```text
Agent 1 - N Task
Session 1 - N Task
Session 1 - N Event
Task 1 - N Event
Agent 1 - N Event
```

## 4. 表结构

## 4.1 agents

保存 Agent 配置。

```sql
CREATE TABLE agents (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  role TEXT NOT NULL,
  description TEXT,
  provider TEXT NOT NULL,
  model TEXT NOT NULL,
  api_key_ref TEXT,
  system_prompt TEXT NOT NULL,
  tools_json TEXT NOT NULL DEFAULT '[]',
  color TEXT,
  avatar TEXT,
  max_tokens INTEGER,
  max_cost_usd REAL,
  max_runtime_seconds INTEGER,
  enabled INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

字段说明：

| 字段 | 说明 |
|---|---|
| `id` | Agent ID，如 `agent_researcher` |
| `name` | 展示名称 |
| `role` | 角色名 |
| `provider` | `anthropic`、`openai`、`gemini` 等 |
| `model` | 模型 ID |
| `api_key_ref` | API Key 引用，不保存明文 |
| `system_prompt` | Agent 系统提示词 |
| `tools_json` | 工具白名单 |
| `max_cost_usd` | 单任务预算 |
| `max_runtime_seconds` | 单任务最长运行时间 |

---

## 4.2 api_keys

只保存引用和元信息，不保存明文。明文放系统 Keychain。

```sql
CREATE TABLE api_keys (
  id TEXT PRIMARY KEY,
  provider TEXT NOT NULL,
  name TEXT NOT NULL,
  keychain_service TEXT NOT NULL,
  keychain_account TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

说明：

```text
SQLite 中只存 keychain_service 和 keychain_account
真正 API Key 明文存 macOS Keychain / Windows Credential Manager / Linux Secret Service
```

---

## 4.3 sessions

保存用户一次任务会话。

```sql
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  user_task TEXT NOT NULL,
  status TEXT NOT NULL,
  final_answer TEXT,
  error TEXT,
  started_at TEXT,
  finished_at TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

---

## 4.4 tasks

保存任务 DAG 中的节点。

```sql
CREATE TABLE tasks (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  agent_id TEXT NOT NULL,
  title TEXT NOT NULL,
  instruction TEXT NOT NULL,
  depends_on_json TEXT NOT NULL DEFAULT '[]',
  status TEXT NOT NULL,
  result TEXT,
  error TEXT,
  started_at TEXT,
  finished_at TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id),
  FOREIGN KEY (agent_id) REFERENCES agents(id)
);
```

索引：

```sql
CREATE INDEX idx_tasks_session_id ON tasks(session_id);
CREATE INDEX idx_tasks_agent_id ON tasks(agent_id);
CREATE INDEX idx_tasks_status ON tasks(status);
```

---

## 4.5 events

最重要的表。所有执行轨迹都在这里。

```sql
CREATE TABLE events (
  id TEXT PRIMARY KEY,
  version TEXT NOT NULL,
  session_id TEXT NOT NULL,
  task_id TEXT,
  agent_id TEXT,
  parent_event_id TEXT,
  type TEXT NOT NULL,
  sequence INTEGER NOT NULL,
  severity TEXT NOT NULL,
  summary TEXT NOT NULL,
  payload_json TEXT NOT NULL,
  created_at TEXT NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id),
  FOREIGN KEY (task_id) REFERENCES tasks(id),
  FOREIGN KEY (agent_id) REFERENCES agents(id)
);
```

索引：

```sql
CREATE UNIQUE INDEX idx_events_session_sequence ON events(session_id, sequence);
CREATE INDEX idx_events_session_id ON events(session_id);
CREATE INDEX idx_events_task_id ON events(task_id);
CREATE INDEX idx_events_agent_id ON events(agent_id);
CREATE INDEX idx_events_type ON events(type);
CREATE INDEX idx_events_created_at ON events(created_at);
```

## 6. 任务依赖模型

`tasks.depends_on_json` 保存数组：

```json
["task_research", "task_outline"]
```

调度器判断：

```text
如果 dependsOn 为空：可立即运行
如果所有 dependsOn 对应任务 status = success：可运行
如果任何依赖 failed：默认不运行，进入 blocked/failed，P1 可支持用户选择继续
```

MVP 可以不新增 `blocked` 状态，依赖失败时将下游任务标记 failed，并写明原因。

## 7. 从事件恢复 UI 状态

加载历史 session 时：

```text
1. 查询 sessions
2. 查询 agents
3. 查询 tasks
4. 查询 events 按 sequence 排序
5. 前端 reduce events 得到当前 WorkbenchState
```

注意：

```text
tasks 表用于快速查询和后端调度
events 表用于 UI 轨迹和状态恢复
二者必须保持一致
```

一致性规则：

```text
1. events 是用户可见轨迹和 UI 恢复依据。
2. sessions/tasks 是调度投影，服务后端快速查询和继续执行。
3. 所有 session/task 状态变更必须通过统一 service 方法完成，不允许业务代码分别手写 UPDATE 和 INSERT event。
4. 如果投影状态和 events 重放结果不一致，UI 以 events 为准，后端调度必须先修复投影再继续运行。
```

## 8. 事务要求

以下操作必须在同一事务中完成：

```text
创建 session + 写 session_started event
创建 task + 写 task_created event
更新 task status + 写对应 event
完成 session + 写 session_finished event
```

推荐封装：

```ts
async function transitionTaskStatus(input: {
  taskId: string;
  nextStatus: TaskStatus;
  eventType: AgentEventType;
  payload: Record<string, unknown>;
}) {
  // BEGIN
  // UPDATE tasks ...
  // INSERT events ...
  // COMMIT
}
```

禁止模式：

```text
先 UPDATE tasks，异步稍后 emit event
先 emit event，失败后再补 UPDATE tasks
同一状态变化在多个模块里重复实现
```

示例：

```text
BEGIN
UPDATE tasks SET status = 'success' WHERE id = ?
INSERT INTO events (... type = 'task_finished' ...)
COMMIT
```

## 9. 数据清理策略

MVP 可以不做自动清理。P1 加：

```text
按 session 删除
保留最近 N 个 session
导出 session trace
清理大 payload
```

如果 tool output 很大，应保存摘要到 events.payload，完整内容保存到 artifacts 表或本地文件。

## 11. 数据库迁移

建议使用迁移文件：

```text

  001_initial_schema.sql
  002_add_tool_calls.sql
  003_add_artifacts.sql
```

应用启动时执行未运行迁移。

```sql
CREATE TABLE schema_migrations (
  version TEXT PRIMARY KEY,
  applied_at TEXT NOT NULL
);
```

MVP 至少需要：

```text
001_initial_schema.sql：agents、api_keys、sessions、tasks、events、settings、schema_migrations
002_seed_default_agents.sql：初始化 5 个默认 Agent
```

## 12. 开发注意事项

```text
1. API Key 明文不能进 SQLite
2. events.payload_json 必须是合法 JSON
3. event.sequence 必须在 session 内递增
4. 删除 session 时要级联或显式删除 tasks/events
5. 不要依赖 events.created_at 排序，优先用 sequence
6. 大内容不要全部塞进 events 表
7. task.status 和事件状态要保持一致
```
